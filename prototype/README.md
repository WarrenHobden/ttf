# Prototype — design workspace

This is where you share design references with me. I use whatever lands here as
the source of truth and turn it into the final master build at the repo root
(`index.html`, `css/`, `js/`).

## How to use it

1. Drop design files into [`designs/`](./designs/) — screenshots, mockups,
   exported PNG/JPG/SVG/PDF, or a written spec (`.md`).
2. Name files so the intent is clear, e.g. `01-home.png`, `02-settings.png`,
   or `flow-onboarding.pdf`. Numbers set the order.
3. Tell me which file(s) to build, or just say "build the latest" and I'll
   work through them.

## What I do with them

- Read the design, then implement it into the root site (the master build).
- Match layout, spacing, colours, and states as closely as plain HTML/CSS/JS allows.
- Ask before guessing on anything ambiguous (fonts, exact colours, interactions).

## Notes

- **`designs/` is the input; the repo root is the output.** I don't ship files
  from here to the live site.
- Formats I can read directly: PNG, JPG, SVG, PDF, and text/Markdown specs.
- For Figma links, paste the URL in chat (I may need the Figma connector authorized).
